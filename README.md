# labor-cost-bot

Телеграм-бот для упрощённого выставления трудозатрат на корпоративном портале.
Бот сам ходит на портал под учёткой сотрудника, показывает, за какие дни ещё не
проставлены трудозатраты, и позволяет заполнить их прямо из чата — распределяя
проценты по проектам через inline-кнопки, без открытия портала в браузере.

## Возможности

- **`/info`** — сводка по трудозатратам: сколько дней осталось заполнить и по
  каким проектам.
- **Режим «Проставить дни»** — интерактивное заполнение прямо в сообщении:
  у каждого проекта свои кнопки `+25 / +50 / -25 / -50`, которыми набираешь его
  долю; дни листаешь стрелками `<-` / `->`. День уходит на портал только когда
  сумма по проектам ровно `100%`.
- **Ежедневное напоминание** — в 18:00 (по умолчанию `Europe/Moscow`) бот пишет в
  чат, сколько дней осталось заполнить, с кнопкой перехода в режим заполнения.
- **Одна учётка / один пользователь** — бот обслуживает только того Telegram-юзера,
  чей ID задан в конфиге; сообщения и колбэки от остальных игнорируются.

## Стек

- Java 25, Spring Boot 4.1
- Spring `RestClient` + Apache HttpClient 5 — HTTP-клиент к порталу
- [pengrad/java-telegram-bot-api](https://github.com/pengrad/java-telegram-bot-api)
  10.1 — long-polling, inline-клавиатуры, колбэки
- Caffeine + Spring Cache — сессия портала, профиль, `data.json`
- Jackson 3 (`tools.jackson`)
- Lombok
- Actuator + Micrometer/Prometheus
- Gradle (Kotlin DSL), сборка через wrapper

## Команды и сценарий

| Команда | Что делает |
|---------|------------|
| `/start`, `/help` | Приветствие + кнопка «Информация по трудозатратам» |
| `/info` | Сводка по трудозатратам; если есть незаполненные дни — кнопка «Проставить дни» |

## Конфигурация


| Переменная | Описание |
|------------|----------|
| `BOT_TOKEN` | Токен Telegram-бота |
| `TG_USER_ID` | Telegram ID единственного разрешённого пользователя |
| `PORTAL_LOGIN` | Логин на портале |
| `PORTAL_PASSWORD` | Пароль на портале |

## Запуск в Docker

Многоступенчатый `Dockerfile` собирает jar в образе с JDK и кладёт его в лёгкий
JRE-образ (запуск от непривилегированного пользователя, том `/data`).

```bash
docker build -t labor-cost-bot .

docker run -d --name labor-cost-bot \
  --env-file .env
  -p 51001:51001 \
  labor-cost-bot
```
